SkyChamp

My story with SkyChamp began when I came to Spirit Bomb Studio. I began my journey in Spirit Bomb by participating the team behind it. My responsibility was to take required features from game designer and implement it. This required me to understand the code base of games and study the new coding style of the company.

Here are the list of features that I did during the first period:
  1. Gacha pets.
  2. Reroll pets skills.
  3. Item fusion.
  4. Fixing bugs (of course).
  5. UI implementation and fix UI layout
After that, I was given the task to implement more technique stuffs such as:
  1. Optimize first loading time remove not-used-yet assets and then load them later

    2. The Unity splash screen takes approximately 2 seconds, followed by our loading screen for 3 seconds, which was deemed excessive for a mobile game. My objective was to streamline the entire system's loading time. Consequently, I opted to minimize the initial loading task, retaining only essential components and relegating low-priority items to subsequent loading stages. This approach proved successful, reducing our loading time to just 1 second, all while maintaining a seamless user experience.

  2. Optimize frame rate by change new bullets to use particle system (ultilize particle system multi-thread)

    3. In the game, bullets often move independently, and to optimize this, we can offload this task to another thread for updates. Unity's particle system is an ideal solution for this scenario. It allows us to track particle positions and provides APIs to control their lifespan and destroy particles as needed.
    The strategy involves creating a list in C# to store instances of a "fake bullet" class, and then performing 2D casting at regular intervals. This approach significantly enhances game performance, especially since the game features a substantial number of bullets being fired by the characters.

    The problems:
    1. Can not use for special bullet (chasing, follow bezier path,...)
    2. Less flexibility: Can not freely control bullet sprite or speed. We have to follow the particle system restrictions
    3. Today, we can have Unity job system that may work better for this case.

  3. Campain script system

    4. The editor and gameplay implementation so that game designer can easily create a campain map where monster have to move in the particular shape or control ship of the user and move it to somewhere after playing a dialog. Using node canvas, I created a fully working editor allow game designers to create and change their level design with ease.

  4. There are more: VFX, AFK Farm, Gacha reroll validation, ...

Atlantis Invaders

When my boss told me I'd be part of the team working on the next version of "SkyChamp," I was thrilled. I felt proud and responsible because we were shaping the future of a beloved game. But there was also some worry because "SkyChamp" was already famous, and we didn't want to mess it up. "Atlantis Invader" gave us a chance to be creative and make the game better while keeping its original spirit.

At the start, we really dug into the game's theme. We asked ourselves, "What story are we trying to tell?" and "What makes players want to go on this virtual adventure?" We talked about this for about a week, and eventually, we came up with an exciting story. In this story, the world has gone through an apocalypse, and the treasures of the old world are hidden deep under the ocean, guarded by ancient AI-controlled ships. In this narrative, players take on the role of a poor mercenary who leads a diverse group of adventurers to find riches by exploring the lost treasures of Atlantis and uncovering the mysteries of our world's history.

While we were developing the game, we used the same engine that powered SkyChamp. I worked on cleaning up and organizing the code to make it better, and then we used this improved code for Atlantis Invaders. My tasks included making the code work smoother, changing how the monsters moved, fixing any remaining issues, and reorganizing some big parts of the code to make it more organized. With these changes, we were all set to create a different version of Atlantis Invaders that was distinct from SkyChamp.

I made the game designer's ideas come alive by adding new ships, bullets, and cool features for both big and small ships. I also put in a drone system and came up with new ways for players to level up. These changes made the game amazing and really great to look at. It turned out to be a fantastic game. I remember the smart ship AI that helped players and the confident shopkeeper who always said buying from him was the best choice.

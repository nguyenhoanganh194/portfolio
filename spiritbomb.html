SkyChamp

My story with SkyChamp began when I came to Spirit Bomb Studio. I began my journey in Spirit Bomb by participating the team behind it. My responsibility was to take required features from game designer and implement it. This required me to understand the code base of games and study the new coding style of the company.

Here are the list of features that I did during the first period:
  1. Gacha pets.
  2. Reroll pets skills.
  3. Item fusion.
  4. Fixing bugs (of course).
  5. UI implementation and fix UI layout
After that, I was given the task to implement more technique stuffs such as:
  1. Optimize first loading time remove not-used-yet assets and then load them later

    2. The Unity splash screen takes approximately 2 seconds, followed by our loading screen for 3 seconds, which was deemed excessive for a mobile game. My objective was to streamline the entire system's loading time. Consequently, I opted to minimize the initial loading task, retaining only essential components and relegating low-priority items to subsequent loading stages. This approach proved successful, reducing our loading time to just 1 second, all while maintaining a seamless user experience.

  2. Optimize frame rate by change new bullets to use particle system (ultilize particle system multi-thread)

    3. In the game, bullets often move independently, and to optimize this, we can offload this task to another thread for updates. Unity's particle system is an ideal solution for this scenario. It allows us to track particle positions and provides APIs to control their lifespan and destroy particles as needed.
    The strategy involves creating a list in C# to store instances of a "fake bullet" class, and then performing 2D casting at regular intervals. This approach significantly enhances game performance, especially since the game features a substantial number of bullets being fired by the characters.

    The problems:
    1. Can not use for special bullet (chasing, follow bezier path,...)
    2. Less flexibility: Can not freely control bullet sprite or speed. We have to follow the particle system restrictions
    3. Today, we can have Unity job system that may work better for this case.

  3. Campain script system

    4. The editor and gameplay implementation so that game designer can easily create a campain map where monster have to move in the particular shape or control ship of the user and move it to somewhere after playing a dialog. Using node canvas, I created a fully working editor allow game designers to create and change their level design with ease.

  4. There are more: VFX, AFK Farm, Gacha reroll validation, ...

Atlantis Invaders

When my boss approached me and shared the exciting news that I would be a key member of the development team for the revolutionary next iterations of "SkyChamp", I couldn't contain my joy. It was a moment of immense pride and responsibility, knowing that my team and I had the opportunity to shape the future of that beloved game. However, there was also a tinge of apprehension. "SkyChamp" had already garnered fame and a dedicated fan base. We were well aware that introducing new features and changes carried a certain level of risk. We didn't want to disrupt the essence of what made these games great in the first place. But with "Atlantis Invader," we found a unique opportunity. This game became the canvas where we could freely unleash our creativity and implement all the exciting ideas we had envisioned for the original "SkyChamp." It was a space to experiment, innovate, and enhance the gaming experience while preserving the integrity of the established "Atlantis Invader" universe.

Initially, we embarked on a thorough exploration of the game's theme. Questions like, "What narrative are we aiming to convey?" and "What drives players to embark on this virtual adventure?" occupied our discussions for an extensive period, spanning perhaps an entire week. Ultimately, our brainstorming efforts crystallized into a compelling storyline set in a post-apocalyptic world. Here, the treasures of the old world lay submerged in the depths of the ocean, zealously guarded by ancient AI-controlled ships. In this narrative, the player assumes the role of a destitute mercenary, leading a motley crew on a quest to amass wealth by delving into the lost treasures of Atlantis and uncovering the mysteries that shroud our world's history.

During the development phase, we leveraged the existing engine from SkyChamp. I dedicated my efforts to refining and refactoring the codebase, which we then employed for Atlantis Invaders. My tasks included optimizing the code, reworking the monster path controller system, addressing lingering bugs, and, ultimately, restructuring large classes to establish a comprehensive code hierarchy. With these improvements in place, we were well-prepared to create a distinct iteration of Atlantis Invaders, setting it apart from SkyChamp.

I brought the game designer's vision to life by introducing new ships, bullets, and innovative mechanics for both motherships and smaller vessels. Additionally, I implemented a drone system and devised fresh leveling-up mechanisms. These enhancements transformed the game into a truly fantastic and visually stunning experience. Indeed, it was a wonderful game. I fondly recall the stubborn ship AI, always eager to guide players on how to play, and the smug shopkeeper character who never failed to remind players that buying from him was the only logical choice.
